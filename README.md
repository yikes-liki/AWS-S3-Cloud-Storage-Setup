*Objective:
The objective of this task is to demonstrate foundational knowledge of cloud computing by manually setting up a secure, scalable cloud storage solution using Amazon Web Services (AWS) Simple Storage Service (S3). The task involves creating a storage bucket, configuring access permissions, and performing basic file operations via the command line.

Scope of Work:
The intern is required to move beyond theoretical knowledge and execute a live cloud deployment. This includes:

Environment Configuration: Setting up the AWS Command Line Interface (CLI) on a local machine and configuring security credentials (IAM) to allow programmatic access to AWS resources.

Resource Provisioning: Creating an S3 bucket (a container for data stored in the cloud) with a globally unique name and specific regional settings.

Security Configuration: Implementing access controls by setting Bucket ACLs (Access Control Lists) and enabling versioning to protect against accidental data overwrites or deletions.

Data Operations: Uploading example files (text and JSON formats) to the newly created bucket and verifying the upload by listing the contents.

Deliverables:
Upon completion, the intern must provide evidence of a fully functional cloud storage system. The specific deliverables include:

A screenshot or log confirming the successful creation of the S3 bucket.

Verification that example files have been uploaded to the cloud bucket.

Confirmation that access permissions are correctly configured (e.g., private access).

Tools & Technologies Used:

Platform: Amazon Web Services (AWS) Free Tier.

Service: S3 (Simple Storage Service).

CLI: AWS Command Line Interface (v2).

Language: Python (Boto3 library) for automation scripting.

Environment: Windows Command Line / PowerShell.

Outcome:
This task simulates a real-world DevOps scenario where an engineer must provision cloud infrastructure as code. By the end of this task, the intern will understand how to navigate the AWS ecosystem, secure cloud assets, and integrate cloud storage into local applications, forming the basis for more complex cloud architectures.*
